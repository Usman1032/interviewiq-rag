Supervised Learning Fundamentals. Supervised learning maps inputs to known outputs using labeled training data. Regression predicts continuous values (e.g. linear regression minimizing mean squared error); classification predicts discrete labels (e.g. logistic regression, decision trees, SVMs). The bias-variance trade-off explains underfitting (high bias, model too simple) versus overfitting (high variance, model memorizes training noise).

Model Evaluation. Accuracy alone is misleading on imbalanced datasets. Precision, recall, and F1 score capture different failure modes -- precision penalizes false positives, recall penalizes false negatives. ROC-AUC summarizes classifier performance across thresholds. Cross-validation (e.g. k-fold) gives a more robust estimate of generalization than a single train/test split.

Neural Networks and Deep Learning. A neural network is a stack of layers applying linear transformations followed by non-linear activations (ReLU, sigmoid, tanh). Backpropagation computes gradients of the loss with respect to weights via the chain rule; gradient descent (or variants like Adam) updates weights to minimize loss. Dropout and batch normalization are common regularization and training-stability techniques.

Retrieval-Augmented Generation (RAG). RAG combines a retrieval system (typically a vector database of embedded document chunks) with a generative language model. At query time, relevant chunks are retrieved via embedding similarity and injected into the model's context so generation is grounded in retrieved facts rather than relying solely on parametric memory. Chunk size, overlap, and embedding model choice materially affect retrieval quality; retrieval quality is often the biggest lever on end-to-end RAG output quality, more so than the generator model itself.

Embeddings and Vector Search. Embeddings map text (or other data) into dense vectors such that semantically similar inputs are close in vector space, typically measured with cosine similarity. Approximate nearest neighbor indexes (e.g. HNSW) trade a small amount of recall for large speedups over exact search at scale.

Transformers and Attention. The transformer architecture relies on self-attention, which lets each token attend to every other token in a sequence, weighted by learned relevance, rather than processing sequentially like RNNs. This parallelism is a major reason transformers scale well with more data and compute. Positional encodings inject sequence order information since attention itself is order-agnostic.

MLOps Considerations. Production ML systems need monitoring for data drift and model drift, not just uptime. Feature stores keep training and serving feature computation consistent. A/B testing or shadow deployment lets teams validate a new model against production traffic before fully cutting over.
